# ApplicationSets — step 1: authoring + parity proof

**Status: INERT. Nothing is wired to ArgoCD. This branch activates nothing.**

This is step 1 of the ApplicationSet migration: author the generators + per-app
configs and *prove* they reproduce the current Application CRs exactly — with
zero cutover, so the parity can be reviewed before anything touches the cluster.

## What this branch adds (purely additive)

- `argo-cd/app-configs/<tier>/<app>.yaml` — **83** tiny per-app config files
  (the data: name, namespace, srcPath, + only the per-app deviations).
- `argo-cd/applicationsets/{platform,security,observability,apps}.yaml` — **4**
  ApplicationSets (git files generator + goTemplate) that template those configs
  into Applications.
- `docs/applicationsets-step1/` — this report + the three proof scripts.

The existing **99 `argo-cd/applications/*.yaml` CRs are untouched**, and
`app-of-apps` still points at `argo-cd/applications/`. The two new directories
(`app-configs/`, `applicationsets/`) are synced by **no** root Application, so
merging this branch changes the cluster in **no** way. It's scaffolding parked
ahead of the (separate, supervised) cutover.

## The parity proof

`extract.py` reads each plain-dir CR and emits a minimal config capturing only
what deviates from the template defaults (asserting invariants so nothing slips
through silently). `render_diff.py` then reconstructs an Application from each
config using the **same logic the ApplicationSet template encodes**, and
**deep-diffs** it against the original CR:

```
PARITY: 83/83 Applications render byte-identical to their CR
```

All three scripts are in this directory; rerun: `python3 extract.py &&
python3 gen-appsets.py && python3 render_diff.py` from a checkout.

### What "byte-identical" covers
name, namespace (incl. the 21 ns≠dir and 3 no-namespace apps), project, source
(repoURL / targetRevision / path / `kustomize` / `directory` blocks verbatim),
destination, `automated.prune`/`selfHeal` (incl. the 3 opt-outs), `syncOptions`
(incl. empty + `CreateNamespace=false` cases), `sync-wave` (6 apps), and the
5-entry `ignoreDifferences` on `codex-refresh`.

## Honest caveat — the one thing I could not run here

The proof above uses a Python renderer that mirrors the ApplicationSet
template + `templatePatch` semantics field-for-field. It does **not** execute
ArgoCD's actual Go templating (the cluster's ArgoCD API wasn't reachable from
where this was built). Because the patch only ever *adds* keys (never conflicts)
and lists are full-replace, strategic-merge and RFC-7386 merge yield the same
result — so the renderer faithfully represents what ArgoCD would produce. Even
so, the **final gate before any cutover is `argocd appset generate` against live
ArgoCD**, diffed against the live Applications. That is step 2, not this.

## Cutover plan (step 2 — separate, supervised, NOT in this branch)

1. `argocd appset generate` each set on live ArgoCD; diff every generated
   Application against the live one until clean. (Live version of the proof above.)
2. **Disable `prune`/`selfHeal` on `app-of-apps`** for the duration.
3. Hand off ownership **without deleting workloads**: orphan the existing
   Applications (`--cascade=orphan`) so each ApplicationSet *adopts* the existing
   `Application` instead of recreating it. Verify Synced/Healthy + no resource
   churn, **tier by tier** (apps → observability → security → platform).
4. Point a root at `argo-cd/applicationsets/` and remove the generated apps'
   old CRs from `argo-cd/applications/` (the 16 explicit CRs stay).
5. Re-enable `prune`/`selfHeal` only once all four projects are adopted + green.

## Design note — why configs live in `argo-cd/app-configs/`, not in each app dir

The proposal pitched co-locating `app-config.yaml` inside each app's manifest
dir ("drop a directory"). But ~58 of the 83 are plain-directory apps: a stray
non-manifest `app-config.yaml` in their synced dir makes ArgoCD error. Putting
the configs in a separate un-synced tree avoids that entirely and keeps parity
exact (no `directory.exclude` workaround). If you prefer the co-located
ergonomics later, it's a small change + a per-app exclude — noted, not done.
